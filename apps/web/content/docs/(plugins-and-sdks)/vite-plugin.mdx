The `@Subvima/vite` plugin automatically creates an Subvima tunnel when your Vite development server starts, giving you a public URL to share your local development environment.

## Installation

```bash
npm install @Subvima/vite
```

```bash
pnpm add @Subvima/vite
```

```bash
yarn add @Subvima/vite
```

## Quick Start

Add the plugin to your Vite configuration:

```ts
// vite.config.ts
import { defineConfig } from 'vite'
import Subvima from '@Subvima/vite'

export default defineConfig({
  plugins: [Subvima()]
})
```

When you run `vite` or `npm run dev`, you'll see your tunnel URL printed alongside the local URL:

```bash
  ➜  Local:   http://localhost:5173/
  ➜  Tunnel:  https://abc123.Subvima.dev
```

## Configuration

The plugin accepts an options object to customize its behavior:

```ts
import { defineConfig } from 'vite'
import Subvima from '@Subvima/vite'

export default defineConfig({
  plugins: [
    Subvima({
      subdomain: 'my-app',
      apiKey: process.env.SUBVIMA_API_KEY,
    })
  ]
})
```

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `subdomain` | `string` | — | Request a specific subdomain for your tunnel URL. Requires authentication. |
| `customDomain` | `string` | — | Use a custom domain. Must be configured in the Subvima dashboard first. |
| `apiKey` | `string` | `process.env.SUBVIMA_API_KEY` | API key for authentication. |
| `serverUrl` | `string` | `wss://api.Subvima.dev/` | Subvima server WebSocket URL. Only change this for self-hosted instances. |
| `enabled` | `boolean` | `process.env.SUBVIMA_ENABLED !== "false"` | Enable or disable the tunnel. |
| `silent` | `boolean` | `false` | Suppress tunnel status logs. |
| `onTunnelReady` | `(url: string) => void` | — | Callback fired when tunnel is successfully established. |
| `onError` | `(error: Error) => void` | — | Callback fired when tunnel encounters an error. |
| `onClose` | `() => void` | — | Callback fired when tunnel connection is closed. |
| `onReconnecting` | `() => void` | — | Callback fired when tunnel is attempting to reconnect. |

## Environment Variables

The plugin respects the following environment variables:

| Variable | Description |
|----------|-------------|
| `SUBVIMA_API_KEY` | API key for authentication (fallback for `apiKey` option) |
| `SUBVIMA_SUBDOMAIN` | Subdomain to use (fallback for `subdomain` option) |
| `SUBVIMA_SERVER_URL` | Server URL (fallback for `serverUrl` option) |
| `SUBVIMA_ENABLED` | Set to `"false"` to disable the tunnel |

## Examples

### Custom Subdomain

Reserve a consistent subdomain for your project:

```ts
import { defineConfig } from 'vite'
import Subvima from '@Subvima/vite'

export default defineConfig({
  plugins: [
    Subvima({
      subdomain: 'my-app',
      apiKey: process.env.SUBVIMA_API_KEY,
    })
  ]
})
```

### Custom Domain

Use your own domain for the tunnel:

```ts
import { defineConfig } from 'vite'
import Subvima from '@Subvima/vite'

export default defineConfig({
  plugins: [
    Subvima({
      customDomain: 'dev.example.com',
      apiKey: process.env.SUBVIMA_API_KEY,
    })
  ]
})
```

### Conditional Enabling

Only enable the tunnel in certain environments:

```ts
import { defineConfig } from 'vite'
import Subvima from '@Subvima/vite'

export default defineConfig({
  plugins: [
    Subvima({
      enabled: process.env.EXPOSE_TUNNEL === 'true',
    })
  ]
})
```

### With Callbacks

React to tunnel events in your application:

```ts
import { defineConfig } from 'vite'
import Subvima from '@Subvima/vite'

export default defineConfig({
  plugins: [
    Subvima({
      onTunnelReady: (url) => {
        console.log(`Share this URL: ${url}`)
        // Copy to clipboard, send notification, etc.
      },
      onError: (error) => {
        console.error('Tunnel error:', error.message)
      },
      onReconnecting: () => {
        console.log('Connection lost, reconnecting...')
      },
    })
  ]
})
```

### Silent Mode

Disable all tunnel logs:

```ts
import { defineConfig } from 'vite'
import Subvima from '@Subvima/vite'

export default defineConfig({
  plugins: [
    Subvima({
      silent: true,
      onTunnelReady: (url) => {
        // Handle the URL silently
      },
    })
  ]
})
```

## Vite Compatibility

The plugin supports Vite versions 4.x, 5.x, 6.x, and 7.x.

## Troubleshooting

### Tunnel not starting

- Ensure your Vite dev server is running on a TCP port (not a Unix socket)
- Check that `SUBVIMA_ENABLED` is not set to `"false"`
- Verify your API key is valid if using a reserved subdomain

### Authentication errors

- Run `Subvima login` in your terminal to authenticate
- Or set the `SUBVIMA_API_KEY` environment variable

### Connection issues

- Check your internet connection
- Verify the server URL is correct (default: `wss://api.Subvima.dev/`)
- The plugin will automatically attempt to reconnect on connection loss

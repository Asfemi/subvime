The `@Subvima/express` plugin automatically creates an Subvima tunnel when your Express development server starts, giving you a public URL to share your local development environment.

## Installation

```bash
npm install @Subvima/express
```

```bash
pnpm add @Subvima/express
```

```bash
yarn add @Subvima/express
```

## Quick Start

Add the plugin to your Express application:

```js
// index.js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

// Apply Subvima middleware
Subvima(app);

app.listen(3000, () => {
  console.log('Server running on port 3000');
});
```

When you run your Express server in development mode, you'll see your tunnel URL:

```bash
Server running on port 3000
  ➜  Tunnel:  https://abc123.Subvima.dev
```

## Configuration

The plugin accepts an options object to customize its behavior:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app, {
  subdomain: 'my-api',
  apiKey: process.env.SUBVIMA_API_KEY,
});

app.listen(3000);
```

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `subdomain` | `string` | — | Request a specific subdomain for your tunnel URL. Requires authentication. |
| `customDomain` | `string` | — | Use a custom domain. Must be configured in the Subvima dashboard first. |
| `apiKey` | `string` | `process.env.SUBVIMA_API_KEY` | API key for authentication. |
| `serverUrl` | `string` | `wss://api.Subvima.dev/` | Subvima server WebSocket URL. Only change this for self-hosted instances. |
| `enabled` | `boolean` | `process.env.SUBVIMA_ENABLED !== "false"` | Enable or disable the tunnel. |
| `silent` | `boolean` | `false` | Suppress tunnel status logs. |
| `onTunnelReady` | `(url: string) => void` | — | Callback fired when tunnel is successfully established. |
| `onError` | `(error: Error) => void` | — | Callback fired when tunnel encounters an error. |
| `onClose` | `() => void` | — | Callback fired when tunnel connection is closed. |
| `onReconnecting` | `() => void` | — | Callback fired when tunnel is attempting to reconnect. |

## Environment Variables

The plugin respects the following environment variables:

| Variable | Description |
|----------|-------------|
| `NODE_ENV` | Set to `development` to enable the tunnel (production is disabled by default) |
| `SUBVIMA_API_KEY` | API key for authentication (fallback for `apiKey` option) |
| `SUBVIMA_SUBDOMAIN` | Subdomain to use (fallback for `subdomain` option) |
| `SUBVIMA_SERVER_URL` | Server URL (fallback for `serverUrl` option) |
| `SUBVIMA_ENABLED` | Set to `"false"` to disable the tunnel |

## Examples

### Custom Subdomain

Reserve a consistent subdomain for your API:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app, {
  subdomain: 'my-api',
  apiKey: process.env.SUBVIMA_API_KEY,
});

app.get('/api/health', (req, res) => {
  res.json({ status: 'ok' });
});

app.listen(3000);
```

### Custom Domain

Use your own domain for the tunnel:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app, {
  customDomain: 'api.example.com',
  apiKey: process.env.SUBVIMA_API_KEY,
});

app.listen(3000);
```

### Conditional Enabling

Only enable the tunnel in certain environments:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app, {
  enabled: process.env.EXPOSE_TUNNEL === 'true',
});

app.listen(3000);
```

### With Callbacks

React to tunnel events in your application:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app, {
  onTunnelReady: (url) => {
    console.log(`API accessible at: ${url}`);
    // Send to Slack, update config, etc.
  },
  onError: (error) => {
    console.error('Tunnel error:', error.message);
  },
  onReconnecting: () => {
    console.log('Connection lost, reconnecting...');
  },
});

app.listen(3000);
```

### Silent Mode

Disable all tunnel logs:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app, {
  silent: true,
  onTunnelReady: (url) => {
    // Handle the URL silently
  },
});

app.listen(3000);
```

### Dynamic Port

Works perfectly with dynamically assigned ports:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app);

// Use port 0 for a random available port
const server = app.listen(0, () => {
  const port = server.address().port;
  console.log(`Server running on port ${port}`);
});
```

## REST API Integration

Perfect for exposing your local API for webhook testing:

```js
import express from 'express';
import Subvima from '@Subvima/express';

const app = express();

Subvima(app, {
  subdomain: 'webhook-test',
  onTunnelReady: (url) => {
    console.log(`Webhook endpoint: ${url}/webhooks`);
  },
});

app.use(express.json());

app.post('/webhooks', (req, res) => {
  console.log('Received webhook:', req.body);
  res.json({ received: true });
});

app.listen(3000);
```

## TypeScript Support

The plugin includes full TypeScript definitions:

```ts
import express, { Application } from 'express';
import Subvima, { SubvimaPluginOptions } from '@Subvima/express';

const app: Application = express();

const options: SubvimaPluginOptions = {
  subdomain: 'my-api',
  apiKey: process.env.SUBVIMA_API_KEY,
  onTunnelReady: (url: string) => {
    console.log(`Tunnel ready: ${url}`);
  },
};

Subvima(app, options);

app.listen(3000);
```

## Middleware Stack

The Subvima plugin doesn't interfere with your middleware stack. You can place it anywhere:

```js
import express from 'express';
import Subvima from '@Subvima/express';
import cors from 'cors';
import helmet from 'helmet';

const app = express();

// Apply Subvima before or after other middleware
Subvima(app);

app.use(cors());
app.use(helmet());
app.use(express.json());

app.listen(3000);
```

## Express Compatibility

The plugin supports Express versions 4.x and 5.x.

## How It Works

The plugin hooks into Express's `app.listen()` method to:

1. Detect when your server starts listening
2. Capture the port number (even for dynamic ports)
3. Establish a WebSocket tunnel to Subvima servers
4. Proxy all incoming HTTP requests to your local server

## Troubleshooting

### Tunnel not starting

- Ensure `NODE_ENV` is set to `development`
- Check that `SUBVIMA_ENABLED` is not set to `"false"`
- Verify your server is listening on a TCP port (not a Unix socket)

### Authentication errors

- Run `Subvima login` in your terminal to authenticate
- Or set the `SUBVIMA_API_KEY` environment variable

### Connection issues

- Check your internet connection
- Verify the server URL is correct (default: `wss://api.Subvima.dev/`)
- The plugin will automatically attempt to reconnect on connection loss

### Not seeing tunnel URL

Make sure you're running in development mode:

```bash
NODE_ENV=development node app.js
```

Or add it to your package.json:

```json
{
  "scripts": {
    "dev": "NODE_ENV=development node app.js"
  }
}
```
